# ShiftControl MVP: система автоматического учета смен

Это MVP (минимально жизнеспособный продукт) системы, которая автоматически обрабатывает сообщения из Telegram-группы и формирует табель смен в Google Sheets.

Проект демонстрирует полный рабочий процесс: от получения сообщения до записи смены в таблицу. Решение может быть доведено до production при добавлении мониторинга, безопасности и обработки ошибок.

## Что делает система

1. Сотрудник отправляет сообщение в Telegram (обычно фото с подписью).
2. Сообщение обрабатывается ботом и передается в очередь задач.
3. Система с помощью AI анализирует сообщение и определяет, является ли оно сменой.
4. Если вероятность высокая — создаётся запись о смене.
5. После подтверждения (или автоматически) данные записываются в Google Sheets.
6. Менеджер может просматривать и подтверждать смены через telegram бота.

Таким образом, система заменяет ручное ведение таблиц и снижает вероятность ошибок.

## Основные возможности

- Настройка и администрирование через Telegram бота
- Автоматическое распознавание смен из сообщений
- Хранение данных о сотрудниках, группах и объектах
- Интеграция с Google Sheets для ведения табеля, включая обратную связь(при изменении данных в таблице, обновляется база данных)
- Очереди задач для обработки сообщений
- API для управления

## Технологии
- Python 3.10+ (asyncio, FastAPI, aiogram, SQLAlchemy)
- PostgreSQL
- Google Sheets API
- Docker
- Redis (для очередей задач)
- OpenAI API (для анализа сообщений)
- Telegram Bot API

## Запуск проекта

1. Клонируйте репозиторий и перейдите в папку проекта:
   ```bash
   git clone
    cd ShiftControl
    ```
2. Установите Docker и Docker Compose, если они ещё не установлены.
3. Создайте файл `.env` в корне проекта и добавьте необходимые переменные окружения (пример ниже):


```ini
# Telegram
BOT_TOKEN=example_bot_token
BOT_PROXY_URL=http://example-proxy:3128
CORE_SERVER_URL=http://core:8000

# Celery
CELERY_BROKER_URL=redis://redis:6379/0
CELERY_RESULT_BACKEND=redis://redis:6379/0

# Postgres
POSTGRES_USER=example_user
POSTGRES_PASSWORD=example_password
POSTGRES_DB=example_db
POSTGRES_PORT=5432
PGADMIN_PORT=8080

# Google Sheets
GOOGLE_SHEETS_CREDENTIALS_JSON=/app/example_credentials.json
GOOGLE_SHEET_NAME=ExampleSheet

# OpenAI
OPENAI_API_KEY=example_api_key

# Администратор
ROOT_ADMIN_USERNAME=example_admin

```


4. Запустите проект:
   ```bash
   docker-compose up --build
   ```


## Что дальше?

- Добавить мониторинг и алерты (например, через Prometheus + Grafana)

- организовать unit/integration тесты
- Уйти от Google Sheets в сторону полноценного фронтенда для управления сменами(CRM, с возможностью экспорта в Excel и др. форматы)

- Реализовать более удобное добавление сотрудников и групп, например, через импорт из CSV или интеграцию с HR-системой.
- Улучшить UX Telegram бота, добавить Telegram Mini Apps для более удобного взаимодействия.
- Добаить сохранение ручного ввода смены в случае, если AI не смог распознать её из сообщения, чтобы не терять данные для дальнейшего дообучения модели.
